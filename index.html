Mars Notebooks

Literate programming framework for Shell Scripts

Mars Notebooks are to Shell scripts, what Jupyter Notebooks are to Python scripts.

Shell scripts are easy to write, nearly impossible to read later.

Mars Notebooks provide a way to write shell scripts that are still decipherable six months after you’ve written them.

ShellFn

Kotlin code as first class shell functions

Shell scripts are inadequate for expressing complex logic, something that statically typed languages such as Kotlin excel at.

What prevents Kotlin from being used for shell scripting is the pain of using it. One has to package it as a JAR file and then run it via java -jar .... There is no command-line autocompletion to nudge your memory about required params, and logic to parse arguments and handle standard streams (stdin, stdout, stderr) is repeated for each script.

ShellFn provides a way to assign a shell alias to a Kotlin function, such as my-grep, and then invoking it on the command line directly using the alias without referencing the containing JAR. Common concerns such as argument parsing, stream pumping, and CLI autocompletion are handled using a conventions over configuration approach.

Logsmash

Smash them log files

Logsmash is a high-performance log processing framework that’s an alternative to frameworks like Elastic Logstash.

Its USP is high performance and a programmatic model as opposed to the configuration driven model used in other frameworks. The log ingestion pipeline is written as code using a Kotlin DSL rather than a configuration document.

This comes with all the benefits of configuration as code such as:

  • Unit testable pipelines
  • IDE based debugging
  • Custom functionality without need to write plugins
